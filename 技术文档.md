# 简历大师（ResumeCraft）技术文档

文档版本：v2.0  
更新日期：2026-04-07  
适用范围：当前仓库代码实现

## 1. 技术栈

1. 前端框架：React 18 + TypeScript
2. 构建工具：Vite 5
3. 状态管理：Zustand
4. 样式体系：Tailwind CSS + 自定义全局 CSS
5. 拖拽排序：@dnd-kit/core / sortable / utilities
6. 富文本编辑：Tiptap（StarterKit + Underline + Link）
7. PDF 导出：后端服务（Go + chromedp）
8. 数据安全：DOMPurify（富文本预览 HTML 清洗）

## 3. 架构说明

### 3.1 页面入口

1. App 根据 `window.location.pathname` 判断普通编辑页或独立预览页。
2. 普通编辑页使用 `AppShell` 三栏布局。
3. `/preview` 路径使用 `PreviewPage` 只渲染分页简历。

### 3.2 三栏职责

1. 左栏（LeftPanel）：模块清单管理（排序、显隐、增删、激活）。
2. 中栏（CenterPanel）：A4 比例实时预览 + 缩放显示。
3. 右栏（RightPanel）：模块表单编辑、模板/样式设置、导出操作。

### 3.3 数据流

1. 所有简历数据集中在 `useResumeStore`。
2. 表单组件调用 store action 更新模块数据。
3. 预览组件从 store 读取数据并渲染模板。
4. 更新后通过防抖写入 localStorage。

## 4. 数据模型

核心类型定义见 `src/types/resume.ts`。

关键对象：
1. Resume：简历根对象（模板、主题色、样式设置、模块数组）。
2. Module：模块对象（id/type/title/visible/data）。
3. PersonalData：个人信息（含头像、头像形状、联系方式、扩展字段）。
4. ResumeStyleSettings：字体、字号、颜色、边距、间距等全局样式参数。

## 5. 状态管理与持久化

实现位置：`src/store/resumeStore.ts`

1. store 使用 Zustand 创建。
2. 初始数据由 `createDefaultResume()` 生成。
3. 持久化 key：`resumecraft_draft`。
4. 持久化版本：`STORAGE_VERSION = 1`。
5. 防抖写入：500ms（`debouncedSave`）。
6. 启动加载：`loadFromStorage()` 恢复草稿并合并默认样式配置。

主要 actions：
1. `setTemplate`
2. `setThemeColor`
3. `setStyleSettings`
4. `updateModuleData`
5. `addModule`
6. `removeModule`
7. `reorderModules`
8. `toggleModuleVisible`
9. `loadFromStorage`

## 6. 预览渲染与分页

### 6.1 模板渲染

`ResumePreview` 根据 `resume.template` 分派：
1. ClassicTemplate
2. ModernTemplate
3. MinimalTemplate

### 6.2 A4 分页算法

实现位置：`src/components/resume/PagedResumePaper.tsx`

关键常量（当前）：
1. `A4_WIDTH_PX = 595`
2. `A4_HEIGHT_PX = 842`
3. `PAGE_TOP_SAFE_PX = 15`
4. `PAGE_BOTTOM_SAFE_PX = 10`
5. `MIN_PAGE_SLICE_PX = 220`

机制：
1. 在隐藏容器中渲染完整简历。
2. 读取 `data-page-break-candidate` 作为可断点候选。
3. 计算每页起始 offset，避免模块被粗暴截断。
4. 同时产出可见分页层和导出分页层（`#resume-paper-export` + `data-export-page`）。

## 7. PDF 导出实现

实现位置：`src/hooks/useExportPDF.ts`

流程：
1. 定位导出根节点（`elementId`，当前调用为 `resume-paper-export`）。
2. 等待字体与图片资源可用。
3. 克隆节点并修正隐藏样式，构造完整 HTML 文档。
4. 前端调用后端接口 `/api/pdf/export` 提交 `{ html, filename }`。
5. 后端使用 Chromium（chromedp）渲染并返回 PDF 二进制。
6. 前端接收 Blob 并触发下载。

## 8. 编辑器与交互细节

### 8.1 富文本

实现位置：`src/components/common/RichTextEditor.tsx`

1. Link 扩展配置 `openOnClick: false`。
2. 链接设置使用应用内弹窗（非浏览器原生 prompt）。
3. 支持快捷输入与工具栏状态同步。

### 8.2 标签输入

实现位置：`src/components/common/TagInput.tsx`

1. 每个实例使用独立 `ref`，避免多实例焦点串扰。
2. 支持分隔符：空格、`,`、`，`、`、`、`+`、回车。
3. 去重后写回数组。

### 8.3 自动保存提示

实现位置：`src/components/layout/RightPanel.tsx`

1. 监听 `lastSavedAt`。
2. 顶部左侧提示显示 2 秒后消失。
3. 使用绝对定位，不挤压操作按钮。

## 9. 样式体系

1. 全局样式在 `src/index.css`。
2. 简历预览字体通过 CSS 变量统一：`--resume-font-family`、`--resume-text-color`、`--resume-font-scale`。
3. 通过映射规则对 `text-[xpt]` 类做统一缩放，保证字号随设置联动。
4. 链接、列表、日期组件等在全局样式中有专项补丁。

## 10. 已知问题与建议

1. 导出链路依赖后端可用性，后端不可用时导出会失败。
2. 可考虑后续引入服务端存储与多简历版本管理。

## 11. 构建与运行

1. 安装依赖：`npm install`
2. 本地开发：`npm run dev`
3. 生产构建：`npm run build`
4. 产物预览：`npm run preview`
