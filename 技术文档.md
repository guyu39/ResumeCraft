## 1. 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + TypeScript 5 + Vite 5 |

| 拖拽 | @dnd-kit/core / sortable / utilities |

## 3. API 客户端架构

### 3.1 客户端封装

实现位置：`src/api/client.ts`

- 封装 `fetch` + 统一错误处理（ApiError）
- 自动注入 Authorization header
- 401 时自动尝试 refresh token 重试
- 导出 `ApiError`、`getToken`、`setTokens`、`clearTokens`、`apiClient`

### 3.2 Token 管理

`src/api/client.ts` 导出：
- `getToken()` — 获取 accessToken
- `setTokens(accessToken, refreshToken)` — 保存 tokens
- `clearTokens()` — 清除 tokens

`src/api/auth.ts` 导出（功能重复，建议清理）：
- `getAccessToken()` — 同 `getToken()`
- `getRefreshToken()` — 获取 refreshToken
- `isAuthenticated()` — 检查是否已登录

> ⚠️ 注意：`auth.ts` 中 `getAccessToken` / `getRefreshToken` 与 `client.ts` 中同名函数功能重复，建议后续统一使用 `client.ts` 导出。

### 3.3 API 模块

| 文件 | 职责 |
|------|------|
| `client.ts` | HTTP 客户端、Token 管理、错误定义 |
| `types.ts` | API 类型定义（与后端契约对齐） |
| `auth.ts` | 认证接口（登录/注册/刷新/登出/me） |
| `resume.ts` | 简历 CRUD（list/create/get/update/delete） |
| `export.ts` | 导出任务（create/getStatus） |
| `ai.ts` | AI 配置/对话/评估/润色（SSE 流式） |

## 4. 状态管理

### 4.1 认证状态

实现位置：`src/store/authStore.ts`

- 使用 Zustand + persist
- 持久化 key：`resumecraft_auth`
- 存储内容：user、isAuthenticated

### 4.2 简历状态

实现位置：`src/store/resumeStore.ts`

- 使用 Zustand + 手动 localStorage 持久化（不使用 persist 中间件）
- 持久化 key：`resumecraft_draft`（单份草稿）、`resumecraft_resume_collection`（简历集合）
- 防抖写入：500ms
- 启动加载：`loadFromStorage()`

主要 actions：`initResume`、`setTemplate`、`setThemeColor`、`setStyleSettings`、`updateModuleData`、`addModule`、`removeModule`、`reorderModules`、`toggleModuleVisible`、`loadFromStorage`、`setResumeTitle`

导出工具函数：`removeResumeFromStorageCollection`、`selectResumeForEditingInStorage`、`saveResumeToCollectionStorage`、`renameResumeInStorage`、`setCurrentResumeIdToStorage`、`getAllResumesFromStorage`

## 5. PDF 导出

### 5.1 当前实现：异步任务模式

实现位置：`src/hooks/useExportPDF.ts` — `useExportPDF()`

流程：
1. POST `/api/resumes/:id/exports` 创建导出任务
2. 轮询 GET `/api/exports/:taskId`（间隔 1.5s，最多 40 次）
3. 任务成功后通过 `task.downloadUrl` 下载

### 5.2 已废弃：同步导出模式

`src/hooks/useExportPDF.ts` — `useSyncExport()`（标记为 @deprecated）

- 调用 POST `/api/pdf/export` 直接渲染 HTML 返回 PDF
- 后端 handler：`backend/internal/handler/pdf.go`
- 该模式保留仅用于兼容旧版前端，未来计划移除

## 6. 云端同步

实现位置：`src/hooks/useCloudSync.ts`

策略：
1. 编辑过程中实时保存到 localStorage（resumeStore 防抖处理）
2. 定期保存到云端（每 30 秒，有变化时）
3. 离开编辑页时静默自动保存
4. 进入页面时查询云端时间戳，不一致则保存

关键 Refs：
- `cloudIdRef` — 云端 ID（UUID）
- `hasSyncedOnMountRef` — 标记是否已完成进入时的同步检查
- `lastSyncedDataRef` — 上次同步的简历数据快照（用于比较变化）

暴露全局方法供外部调用：
- `window.__cloudSyncSetCloudId(id)` — 设置云端 ID
- `window.__cloudSyncMarkSynced()` — 标记为已完成初始同步

## 7. 已废弃代码清单

| 文件 | 废弃内容 | 替代方案 |
|------|---------|----------|
| `src/hooks/useExportPDF.ts` — `useSyncExport()` | 同步 PDF 导出 | `useExportPDF()` 异步任务模式 |
| `backend/internal/handler/pdf.go` | 同步渲染接口 `/api/pdf/export` | 异步任务流程 |
| `src/api/auth.ts` — `getAccessToken/getRefreshToken/isAuthenticated` | 与 client.ts 功能重复 | 使用 `client.ts` 导出的 `getToken/setTokens/clearTokens` |
| `src/hooks/useExportPDF.ts` 中未使用的 `waitForFontsReady`/`waitForImagesReady` | 同步导出专用 | 异步任务模式不使用 |

## 8. 已知问题与待办

1. **简历名称冲突循环**：同步成功后本地简历未删除，`hasSyncedOnLogin` 未持久化，刷新后重复触发同步。
2. **版本管理 UI 缺失**：后端已支持 `RestoreVersion`，前端仅有底层实现，列表/恢复 UI 待开发。
3. **导出文件下载接口**：`GET /api/exports/:taskId/download` 后端尚未实现（当前通过 `/api/exports/:taskId` 返回 `downloadUrl`）。
4. **client.ts 与 auth.ts Token 函数重复**：`auth.ts` 导出的 `getAccessToken`/`getRefreshToken` 与 `client.ts` 导出的 `getToken` 功能重叠，建议统一。

## 9. 构建与运行

### 前端

```bash
npm install
npm run dev      # 开发服务器（API 代理到 :8787）
npm run build    # 生产构建
npm run preview  # 预览构建产物
```

### 后端

```bash
cd backend
go mod tidy
go run ./cmd/server   # 默认 :8787
```

### 环境变量

前端（Vite）：`VITE_API_BASE_URL=http://localhost:8787/api`

后端：
- `DATABASE_URL=postgres://user:password@localhost:5432/resumecraft?sslmode=disable`
- `JWT_SECRET=your-secret-key`
- `AI_*`（可选）
