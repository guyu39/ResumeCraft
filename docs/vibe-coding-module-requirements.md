# Vibe Coding 能力模块 — 需求文档

## 1. 背景与目标

### 1.1 背景

当前大厂（字节、腾讯、阿里、百度等）技术岗位面试与绩效考核中，**Vibe Coding（AI 辅助研发能力）** 已成为独立考核维度。传统的"专业技能"和"自我评价"模块无法结构化呈现候选人在 AI 驱动研发全流程中的量化成果。

### 1.2 目标

在简历编辑器中新增「**AI 工程**」模块，使用户能够：

1. 按 10 大考核维度结构化填写 AI 辅助研发的落地成果。
2. 每项指标支持量化数据（工时节省、缺陷率变化等）与佐证材料链接。
3. 在预览和导出 PDF 中专业呈现，直接面向大厂面试官。

## 2. 模块定义

### 2.1 基本信息

| 属性 | 值 |
|------|-----|
| 模块标识 | `ai-engineering` |
| 模块类型名 | `AI 工程` / `AI Engineering`（中/英） |
| 位置 | 所有固定模块和可选模块之后（`custom` 模块之前） |
| 最大数量 | 1（唯一模块，不可重复添加） |
| 可见性 | 默认可见，可切换隐藏 |

### 2.2 数据模型

```typescript
interface AIEngineeringData {
  // 10 大指标，每项一个条目
  indicators: AIEngineeringIndicator[]
}

interface AIEngineeringIndicator {
  id: string                          // UUID
  category: AICategory                // 指标分类（10 选 1）
  scenario: string                    // 落地场景（绑定具体系统/模块）
  description: string                 // 方案描述（rich text）
  quantMetrics: AIQuantMetric[]       // 量化指标（可多条）
  tools: string[]                     // 使用的 AI 工具链
  evidenceLinks: string[]             // 佐证链接/文档
  timeRange: string                   // 落地时间段（如 "2025.06 - 至今"）
}

interface AIQuantMetric {
  label: string                       // 指标名（如 "工时节省"）
  value: string                       // 量化值（如 "60%"）
  baseline: string                    // 基线（如 "改造前 5人日，改造后 2人日"）
}

type AICategory =
  | 'planning'          // 指标1：AI驱动工作规划
  | 'task-modeling'     // 指标2：AI任务建模与引导
  | 'output-validation' // 指标3：AI输出理解校验
  | 'toolchain'         // 指标4：AI工具链与能力体系
  | 'feasibility'       // 指标5：AI方案可行性落地
  | 'workflow'          // 指标6：AI工作流应用
  | 'efficiency'        // 指标7：效率提升
  | 'quality'           // 指标8：质量与风险控制
  | 'scale'             // 指标9：成果规模化推广
  | 'innovation'        // 指标10：AI创新能力
```

### 2.3 10 大指标详细说明

#### 指标 1：AI 驱动工作规划能力（planning）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 绑定到具体模块/功能（如"2025Q3 重构用户中心鉴权模块"） |
| 方案描述 | 包含技术选型、架构实现路径 |
| 量化指标 | 排期精确到周/日，验收标准可验证 |
| 工具链 | 规划阶段使用的 AI 工具 |
| 佐证 | PRD 文档、排期甘特图、技术方案评审记录 |

#### 指标 2：AI 任务建模与引导能力（task-modeling）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 拆解的模糊需求（如"优化订单列表加载慢"） |
| 方案描述 | 如何定义输入/输出/边界/依赖；补充了什么上下文（代码/接口/日志/规范） |
| 量化指标 | 分阶段引导的轮次数、纠偏次数、最终收敛耗时 |
| 工具链 | 使用的 Prompt 工程策略 |
| 佐证 | 任务拆解文档、验收标准、测试用例、Prompt 模板 |

#### 指标 3：AI 输出理解、校验与判断能力（output-validation）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 对 AI 产出做技术把控的具体案例 |
| 方案描述 | 四类核验（文档/逻辑/数据/回归）的执行方式；识别到的隐藏风险（安全/并发/兼容性/数据一致性） |
| 量化指标 | 发现并修复的隐藏风险数量、方案对比选型节省的成本 |
| 工具链 | 校验工具、测试框架 |
| 佐证 | 风险清单、测试报告、多方案对比记录 |

#### 指标 4：AI 工具链与能力体系熟练度（toolchain）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 编码/重构/排障/文档等不同场景的工具选型 |
| 方案描述 | 如何打通本地工程（代码仓→构建→CI→缺陷系统）；自动化协作链路（检索→生成→校验→回归→发布） |
| 量化指标 | 工具链覆盖的场景数、自动化环节数 |
| 工具链 | Cursor/Copilot/Claude/GPT + 自建工具 |
| 佐证 | 工具使用日志、流水线配置、自定义模板/脚本库 |

#### 指标 5：AI 方案可行性落地能力（feasibility）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 完整 AI 辅助方案上线案例 |
| 方案描述 | 方案设计→评审→开发→测试→上线的完整链路 |
| 量化指标 | 方案通过率、上线数量、落地周期 |
| 佐证 | 上线记录、灰度验证报告 |

#### 指标 6：AI 工作流应用能力（workflow）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 标准化的 AI 工作流案例 |
| 方案描述 | 流程定义（触发→AI处理→校验→人工确认→发布） |
| 量化指标 | 流程复用次数、覆盖的研发场景数 |
| 佐证 | 流程定义文档、SOP、执行记录 |

#### 指标 7：效率提升能力（efficiency）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | AI 提效的具体场景（开发/测试/文档/运维） |
| 方案描述 | 可复现、可迁移、可规模化的提效方案 |
| 量化指标 | **强量化**：工时节省百分比、迭代周期缩短天数、同等工时多产出数量 |
| 佐证 | 前后对比数据、工时统计报表 |

#### 指标 8：质量与风险控制能力（quality）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | AI 辅助质量管控的具体切入点 |
| 方案描述 | CR 评审、自动化测试、质量门禁的 AI 集成方式 |
| 量化指标 | 线上故障率变化、缺陷数变化、回归测试覆盖率 |
| 佐证 | 质量报表、CR 记录、测试覆盖率报告 |

#### 指标 9：成果规模化推广能力（scale）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 将个人 AI 实践推广到团队/部门的案例 |
| 方案描述 | Prompt 库/SOP/Agent 规则/培训文档的建设方式 |
| 量化指标 | 使用人数、覆盖团队数、跨部门范围、周/月调用频次、采纳率 |
| 佐证 | 调用日志、培训记录、版本迭代记录 |

#### 指标 10：AI 创新能力（innovation）

| 字段 | 填写引导 |
|------|---------|
| 落地场景 | 依托 AI 打造的新业务场景或内部增效工具 |
| 方案描述 | 创新点、业务价值、与传统方式的差异 |
| 量化指标 | 新功能/系统产出量、内部增效工具的提效数据 |
| 佐证 | 新功能/系统产出、优化后的流程规范、用户反馈 |

## 3. 前端 UI 设计

### 3.1 模块列表入口

- 左栏模块列表底部，「+ 添加模块」下拉菜单中新增 **「AI 工程」** 选项。
- 已有该模块时选项灰度不可点（最大数量 = 1）。

### 3.2 编辑面板（右栏）

```
┌─ AI 工程 ──────────────────────────────────┐
│  [选择指标分类 ▼]                                │
│                                                   │
│  ┌─ 指标1: AI驱动工作规划 ─────────────────────┐ │
│  │ 落地场景: [__________绑定具体系统/模块______] │ │
│  │ 方案描述: [富文本编辑器_____________________] │ │
```

### 3.3 交互细节

| 交互 | 行为 |
|------|------|
| 指标分类下拉 | 10 项可选，已有指标灰显（每类最多 1 条） |
| 添加指标 | 追加到列表底部，自动聚焦「落地场景」输入框 |
| 删除指标 | 每项指标右上角 × 按钮，hover 显示 |
| 拖拽排序 | 支持指标条目拖拽调整顺序 |
| 量化指标行 | 行内编辑，指标名/值/基线三个输入框，支持添加/删除 |
| 工具/链接 | Tag 式输入，回车添加，点击 x 删除 |
| 富文本编辑 | 「方案描述」使用 Tiptap 编辑器，与现有富文本模块一致 |

## 4. 数据存储

### 4.1 模块注册

在 `MODULE_META_LIST` 中新增：

```typescript
{
  type: 'ai-engineering',
  label: 'AI 工程',
  maxCount: 1,
  removable: true,
}
```

### 4.2 localStorage

与其他模块一致的持久化策略：
- 编辑时防抖保存到 `STORAGE_KEY`。
- 快照草稿保存到 `resumecraft_snapshot_draft_:snapshotId`。

### 4.3 后端数据库

数据存入 `resumes.content` 的 `modules` 数组中，无需新增表。格式：

```json
{
  "id": "uuid",
  "type": "ai-engineering",
  "title": "AI 工程",
  "visible": true,
  "data": {
    "indicators": [
      {
        "id": "uuid",
        "category": "planning",
        "scenario": "...",
        "description": "...",
        "quantMetrics": [{ "label": "工时节省", "value": "60%", "baseline": "5人日→2人日" }],
        "tools": ["Cursor", "Claude"],
        "evidenceLinks": ["https://..."],
        "timeRange": "2025.07 - 2025.09"
      }
    ]
  }
}
```

## 7. 验收标准

1. 可从左栏「+ 添加模块」中添加「AI 工程」模块。
2. 支持添加/编辑/删除/排序 10 类考核指标条目。
3. 每项指标包含：指标分类、落地场景、方案描述（富文本）、量化指标（多条）、AI 工具、佐证链接、时间段。
4. 左栏模块列表中显示为「AI 工程」，可拖拽排序、切换显隐、删除。
5. 三套模板 A4 预览中正确渲染。
6. PDF 导出中正确渲染。
7. 中英文切换时字段标签正确翻译。
8. 快照对比可识别 indicators 数组内的字段级差异。
9. 自动保存 + 云端同步 + 快照草稿正常运作。
