# 二期接口契约草案（含请求/响应 JSON 示例）

文档版本：v0.9（草案）
更新日期：2026-04-10
适用阶段：二期（账号体系 + 云端持久化 + 后端 PDF 导出）

## 1. 通用约定

### 1.1 基础信息

1. Base URL：`/api`
2. 数据格式：`application/json; charset=utf-8`
3. 时间格式：ISO 8601（UTC），例如：`2026-04-10T08:30:00.000Z`
4. 鉴权方式：Bearer Token（Access Token）

请求头示例：

```http
Authorization: Bearer <access_token>
Content-Type: application/json
```

### 1.2 通用响应结构

成功响应：

```json
{
  "code": "OK",
  "message": "success",
  "requestId": "req_01HXYZ...",
  "data": {}
}
```

失败响应：

```json
{
  "code": "RESUME_NOT_FOUND",
  "message": "简历不存在或无权限访问",
  "requestId": "req_01HXYZ...",
  "details": {
    "resumeId": "res_123"
  }
}
```

### 1.3 错误码建议

1. `OK`
2. `BAD_REQUEST`
3. `UNAUTHORIZED`
4. `FORBIDDEN`
5. `NOT_FOUND`
6. `RATE_LIMITED`
7. `RESUME_NOT_FOUND`
8. `EXPORT_TASK_NOT_FOUND`
9. `EXPORT_FAILED`
10. `VALIDATION_ERROR`
11. `INTERNAL_ERROR`

### 1.4 分页约定

查询参数：

1. `page`：从 1 开始
2. `pageSize`：默认 20，最大 100

分页返回示例：

```json
{
  "code": "OK",
  "message": "success",
  "requestId": "req_01",
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 135,
      "totalPages": 7
    }
  }
}
```

## 5. 导出接口（Exports）

### 5.1 创建导出任务

`POST /api/resumes/:id/exports`

请求：

```json
{
  "versionId": "ver_1002",

}
```

```json
{
  "code": "OK",

  "requestId": "req_export_create_01",

    "taskId": "exp_9001",

```

## 6. 推荐状态码

1. `200`：成功。
2. `201`：创建成功。
3. `202`：任务已受理（可用于导出任务创建）。
4. `400`：参数错误。
5. `401`：未认证。
6. `403`：无权限。
7. `404`：资源不存在。
8. `409`：并发冲突（版本冲突）。
9. `429`：触发限流。
10. `500`：服务内部错误。

## 7. 前后端联调要点

1. 前端自动保存建议传 `clientUpdatedAt`，后端可做并发冲突检测。
2. 导出接口必须指定 `versionId`，避免编辑中状态漂移。
3. 下载链接建议短时有效并校验用户归属。
4. 前端在 `PROCESSING` 状态建议 1s-2s 轮询一次，最长轮询 60s 后提示用户后台继续生成。
