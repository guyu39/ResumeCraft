# 简历大师（ResumeCraft）需求文档（一期已上线 + 二期规划）

文档版本：v3.0  
更新日期：2026-04-10  
适用范围：当前前端仓库 + 二期后端建设

## 1. 背景与目标

### 1.1 当前现状（一期）

1. 已具备三栏编辑、实时预览、模板样式调整、AI 评估与建议、前端导出 PDF 能力。
2. 当前导出链路依赖浏览器环境，存在样式一致性和稳定性风险。
3. 当前草稿以本地存储为主，缺少账号登录与跨端同步能力。

### 1.2 二期核心目标

1. 引入登录鉴权，实现“用户身份”与“数据归属”。
2. 引入后端持久化，支持多简历管理、跨端恢复、历史版本。
3. 引入服务端 PDF 导出，解决前端导出在打印策略、字体、样式兼容上的不稳定问题。
4. 在不破坏一期体验前提下，完成平滑迁移与灰度上线。

## 2. 用户与业务场景

1. 求职用户：长期维护多份简历，针对不同岗位快速调整并导出。
2. 高频迭代用户：在不同设备上继续编辑，要求数据实时同步。
3. 强样式依赖用户：要求导出 PDF 与预览尽可能一致，可直接用于投递。

## 3. 版本范围定义

### 3.1 一期范围（已实现）

1. 三栏编辑、模块化管理、模板样式设置。
2. 前端实时预览与分页展示。
3. 前端导出 PDF（客户端链路）。
4. AI 建议与 AI 评估功能。
5. localStorage 自动保存。

### 3.2 二期范围（本次需求）

1. 账号体系：注册、登录、登出、会话续期。
2. 云端持久化：简历 CRUD、自动保存、列表管理、历史版本。
3. 后端 PDF 导出：任务创建、状态查询、文件下载。
4. 安全与审计：访问控制、导出记录、错误追踪。
5. 与一期兼容：保留编辑体验，切换导出入口至后端。

### 3.3 二期非目标

1. 暂不做多人协作实时编辑。
2. 暂不做企业管理员和复杂组织权限。
3. 暂不做支付、订阅、模板商城。

## 4. 功能需求（FR）

### 4.1 登录与账号（FR-AUTH）

1. 支持邮箱 + 密码注册登录。
2. 支持验证码登录或第三方登录作为可扩展项（默认关闭）。
3. 支持会话过期自动续期（Refresh Token）。
4. 支持手动登出并清理本地令牌。
5. 未登录访问编辑页时，允许试用但导出与云同步需登录。

验收口径：
1. 登录成功后可访问用户私有简历列表。
2. Token 过期后可无感续期，不中断编辑。
3. 登出后无法访问私有 API。

### 4.2 简历持久化（FR-DATA）

1. 支持云端简历列表。
2. 支持新建、重命名、删除、复制简历。
3. 支持编辑自动保存到后端（建议 800ms-1500ms 防抖）。
4. 支持版本快照（至少保留最近 20 个版本）。
5. 支持从版本恢复。
6. 支持本地草稿迁移：登录后提示是否导入本地内容。

验收口径：
1. 同账号跨浏览器可看到相同简历列表。
2. 页面刷新后可恢复最近编辑状态。
3. 删除简历后不可在列表中访问该资源。

### 4.3 后端 PDF 导出（FR-PDF）

1. 导出按钮改为调用后端导出接口。
2. 服务端基于标准渲染引擎生成 PDF（建议 Playwright/Chromium printToPDF）。
3. 支持异步任务导出：
	1. 创建导出任务
	2. 轮询/推送任务状态
	3. 成功后下载文件
4. 导出必须基于“指定简历版本”生成，保证可追溯。
5. 导出失败时返回明确错误码与可读提示。

验收口径：
1. 导出文件可稳定打开且页面尺寸为 A4。
2. 同一版本重复导出视觉结果一致。
3. 导出失败可在前端明确提示并支持重试。

### 4.4 权限与安全（FR-SEC）

1. 用户只能访问自己的简历与导出文件。
2. 所有写操作需鉴权。
3. 关键操作记录审计日志：登录、导出、删除、版本恢复。
4. 接口需具备基础限流（尤其登录与导出）。

### 4.5 前端交互调整（FR-UX）

1. 导出入口改为“生成 PDF”任务流程。
2. 显示导出状态：排队中、生成中、成功、失败。
3. 导出成功提供下载与再次导出。
4. 数据保存状态改为“本地已保存/云端已同步/同步失败”。

## 5. 数据与对象模型（二期新增）

### 5.1 核心实体

1. User：用户实体。
2. Resume：简历主实体（当前结构兼容一期）。
3. ResumeVersion：简历版本快照。
4. ExportTask：导出任务。
5. ExportFile：导出文件元数据。

### 5.2 关键字段建议

1. Resume：id、userId、title、content、updatedAt、latestVersionId。
2. ResumeVersion：id、resumeId、versionNo、contentSnapshot、createdAt。
3. ExportTask：id、resumeId、versionId、status、errorCode、errorMessage、createdAt、finishedAt。
4. ExportFile：id、taskId、storageKey、downloadUrl、expiresAt。

## 6. 接口需求（API）

### 6.1 认证接口

1. POST /api/auth/register
2. POST /api/auth/login
3. POST /api/auth/refresh
4. POST /api/auth/logout
5. GET /api/auth/me

### 6.2 简历接口

1. GET /api/resumes
2. POST /api/resumes
3. GET /api/resumes/:id
4. PUT /api/resumes/:id
5. DELETE /api/resumes/:id

### 6.3 版本接口

1. GET /api/resumes/:id/versions
2. POST /api/resumes/:id/versions/:versionId/restore

### 6.4 导出接口

1. POST /api/resumes/:id/exports
2. GET /api/exports/:taskId
3. GET /api/exports/:taskId/download

## 7. 非功能需求（NFR）

### 7.1 性能

1. 普通保存接口 P95 < 500ms。
2. 导出任务创建接口 P95 < 300ms。
3. 单份简历 PDF 生成目标：P95 < 10s（不含排队）。

### 7.2 可用性

1. 导出服务月可用性不低于 99.9%。
2. 导出失败需可重试，且不影响编辑流程。

### 7.3 一致性

1. 预览与导出渲染规则统一。
2. 导出必须绑定版本号，避免“编辑中数据漂移”。

### 7.4 安全

1. 传输全链路 HTTPS。
2. Token 安全存储与过期策略。
3. 导出文件下载链接需时效签名。

## 8. 状态机与流程

### 8.1 导出任务状态

1. QUEUED：已创建。
2. PROCESSING：生成中。
3. SUCCESS：生成完成可下载。
4. FAILED：生成失败可重试。

### 8.2 前端导出流程

1. 点击导出。
2. 创建导出任务。
3. 轮询任务状态（或 SSE/WebSocket 推送）。
4. 成功后展示下载按钮。
5. 失败后展示错误与重试。

## 9. 迁移与兼容策略

1. 二期上线初期保留本地草稿读取能力。
2. 登录后触发一次“本地草稿导入提示”。
3. 导出入口默认走后端，保留前端导出作为灰度兜底开关（仅内部使用）。

## 10. 验收标准（二期）

1. 用户可完成注册、登录、登出、会话续期。
2. 用户可创建并管理多份云端简历。
3. 编辑内容可自动同步到后端并跨端恢复。
4. 导出按钮走后端任务流程，可下载可用 PDF。
5. 同一简历版本重复导出结果一致，不出现随机空白。
6. 未授权用户无法访问他人简历与导出文件。

## 11. 里程碑与分期建议（2 期）

### 第 2 期-A（账号 + 云存储）

1. 账号体系与鉴权中间件。
2. 简历 CRUD 与自动保存。
3. 本地草稿迁移。
4. 基础审计与限流。

### 第 2 期-B（后端导出 + 稳定性）

1. 导出任务系统。
2. 服务端 PDF 渲染与文件存储。
3. 导出状态回传与前端下载链路。
4. 导出监控、失败重试与告警。

## 12. 风险与应对

1. 风险：服务端渲染与前端预览不一致。  
	应对：统一渲染入口与样式基线，固定字体与页面尺寸。

2. 风险：导出并发导致队列堆积。  
	应对：任务队列 + 并发上限 + 优先级控制。

3. 风险：历史数据迁移失败。  
	应对：保留本地兜底、迁移日志与可回滚机制。
