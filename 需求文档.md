# 简历大师（ResumeCraft）需求文档

文档版本：v3.2
更新日期：2026-04-23
适用范围：当前前端仓库 + 后端服务

## 1. 背景与目标

### 1.1 当前现状

1. 已具备三栏编辑、模块化管理、模板样式调整、AI 评估与建议能力。
2. 后端 PDF 导出能力已实现（异步任务模式）。
3. 账号体系与云端持久化已完成，登录后可跨端同步。
4. 本地草稿登录后可导入云端。

### 1.2 后续迭代方向

1. 历史版本管理（列表、恢复）。
2. 导出文件下载接口（当前导出成功后通过 `/api/exports/:taskId/download` 获取文件）。
3. AI 能力后端化，详见 [AI后端移植与双版本规划.md](AI后端移植与双版本规划.md)。
4. 多端实时同步优化。

## 2. 用户与业务场景

1. 求职用户：长期维护多份简历，针对不同岗位快速调整并导出。
2. 高频迭代用户：在不同设备上继续编辑，要求数据实时同步。
3. 强样式依赖用户：要求导出 PDF 与预览尽可能一致，可直接用于投递。

## 4. 功能需求（FR）

### 4.1 登录与账号（FR-AUTH）

1. 支持邮箱 + 密码注册登录。
2. 支持验证码登录或第三方登录作为可扩展项（默认关闭）。
3. 支持会话过期自动续期（Refresh Token）。
4. 支持手动登出并清理本地令牌。
5. 未登录访问编辑页时，允许试用但导出与云同步需登录。

验收口径：
1. 登录成功后可访问用户私有简历列表。
2. Token 过期后可无感续期，不中断编辑。
3. 登出后无法访问私有 API。

### 4.2 简历持久化（FR-DATA）

1. 支持云端简历列表。
2. 支持新建、重命名、删除、复制简历。
3. 支持编辑自动保存到后端（建议 800ms-1500ms 防抖）。
4. 支持版本快照（Create/Update 时自动创建，保留最近 N 个版本）。
5. 支持从版本恢复（后端已实现 RestoreVersion，前端 UI 待开发）。
6. 支持本地草稿迁移：登录后提示是否导入本地内容。

验收口径：
1. 同账号跨浏览器可看到相同简历列表。
2. 页面刷新后可恢复最近编辑状态。
3. 删除简历后不可在列表中访问该资源。

### 4.3 后端 PDF 导出（FR-PDF）

1. 导出按钮改为调用后端导出接口。
2. 服务端基于 Chromium（chromedp）生成 PDF。
3. 支持异步任务导出：
   - 创建导出任务（POST /api/resumes/:id/exports）
   - 轮询任务状态（GET /api/exports/:taskId）
   - 成功后下载文件（GET /api/exports/:taskId/download）
4. 导出必须基于"指定简历版本"生成，保证可追溯。
5. 导出失败时返回明确错误码与可读提示。

验收口径：
1. 导出文件可稳定打开且页面尺寸为 A4。
2. 同一版本重复导出视觉结果一致。
3. 导出失败可在前端明确提示并支持重试。

### 4.4 权限与安全（FR-SEC）

1. 用户只能访问自己的简历与导出文件。
2. 所有写操作需鉴权。
3. 关键操作记录审计日志：登录、导出、删除、版本恢复。
4. 接口需具备基础限流（尤其登录与导出）。

### 4.5 前端交互调整（FR-UX）

1. 导出入口改为"生成 PDF"任务流程。
2. 显示导出状态：排队中、生成中、成功、失败。
3. 导出成功提供下载与再次导出。
4. 数据保存状态改为"本地已保存/云端已同步/同步失败"。

## 5. 数据与对象模型

### 5.1 核心实体

1. User：用户实体。
2. Resume：简历主实体。
3. ResumeVersion：简历版本快照。
4. ExportTask：导出任务。
5. ExportFile：导出文件元数据。

### 5.2 关键字段

1. Resume：id、userId、title、content（jsonb）、updatedAt、latestVersionId。
2. ResumeVersion：id、resumeId、versionNo、contentSnapshot（jsonb）、createdAt。
3. ExportTask：id、resumeId、versionId、status、errorCode、errorMessage、createdAt、finishedAt。
4. ExportFile：id、taskId、storageKey、downloadUrl、expiresAt。

### 5.3 PostgreSQL 落库约束

1. 数据库统一采用 PostgreSQL（建议 15+）。
2. 主键统一采用 UUID，时间字段统一采用 timestamptz。
3. Resume.content 与 ResumeVersion.contentSnapshot 使用 jsonb 存储。
4. 必建索引：resume(user_id, updated_at desc)、resume_version(resume_id, version_no desc)。
5. 必建唯一约束：users(email)、resume_version(resume_id, version_no)。
6. 全部写操作需带 user_id 归属校验，防止越权。

## 7. 非功能需求（NFR）

### 7.1 性能

1. 普通保存接口 P95 < 500ms。
2. 导出任务创建接口 P95 < 300ms。
3. 单份简历 PDF 生成目标：P95 < 10s（不含排队）。

### 7.2 可用性

1. 导出服务月可用性不低于 99.9%。
2. 导出失败需可重试，且不影响编辑流程。

### 7.3 一致性

1. 预览与导出渲染规则统一。
2. 导出必须绑定版本号，避免"编辑中数据漂移"。

### 7.4 安全

1. Token 安全存储与过期策略。
2. 导出文件下载链接需时效签名。

## 8. 状态机与流程

### 8.1 导出任务状态

1. QUEUED：已创建。
2. PROCESSING：生成中。
3. SUCCESS：生成完成可下载。
4. FAILED：生成失败可重试。

### 8.2 前端导出流程

1. 点击导出。
2. 创建导出任务（POST /api/resumes/:id/exports）。
3. 轮询任务状态（GET /api/exports/:taskId，间隔 1.5s）。
4. 成功后展示下载按钮。
5. 失败后展示错误与重试。

## 9. 迁移与兼容策略

1. 登录后触发一次"本地草稿导入确认"。
2. 导出入口默认走后端异步任务，旧版同步导出（/api/pdf/export）已废弃但保留兼容。

## 10. 验收标准

1. 用户可完成注册、登录、登出、会话续期。
2. 用户可创建并管理多份云端简历。
3. 编辑内容可自动同步到后端并跨端恢复。
4. 导出按钮走后端任务流程，可下载可用 PDF。
5. 同一简历版本重复导出结果一致，不出现随机空白。
6. 未授权用户无法访问他人简历与导出文件。

## 12. 风险与应对

1. 风险：服务端渲染与前端预览不一致。
   应对：统一渲染入口与样式基线，固定字体与页面尺寸。

2. 风险：导出并发导致队列堆积。
   应对：任务队列 + 并发上限 + 优先级控制。

3. 风险：历史数据迁移失败。
   应对：保留本地兜底、迁移日志与可回滚机制。

4. 风险：简历名称冲突循环（刷新后重复提示）。
   应对：同步成功后删除本地简历，持久化 `hasSyncedOnLogin` 状态。
